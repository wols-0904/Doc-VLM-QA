# Course-Doc-VLM-QA: 基于动态文档索引与VLM的中文课程文档问答系统

本项目为“多模态大模型原理与应用”课程的期中实践项目。旨在解决传统文本检索系统无法处理课程资料（如讲义、论文、PPT截图）中图表、公式和版面信息的问题。系统采用“直接多模态问答+动态文档索引检索”的轻量架构完成系统搭建。

## 🌟 核心特性

* **动态任务路由**：若用户只上传单张图片或单页截图，系统直接将图片和问题输入VLM进行回答；若上传多页PDF、PPT导出PDF或多个文件，系统先对当前资料构建临时索引。
* **多模态文档理解**：支持处理文本、表格、公式、图像、流程图和实验结果图等多模态元素。
* **精准溯源引用**：多页文档问题的回答中包含证据来源（如引用页码）。
* **轻量化临时索引**：不预先构建一个固定外部知识库，而是针对当前资料动态构建临时文档索引。

## 🛠️ 技术栈

* **文档解析**：PyMuPDF 或 pdfplumber 提取每页文本
* **向量检索**：bge-small-zh-v1.5 + 临时向量索引 FAISS
* **大语言模型 (LLM)**：Qwen2.5-7B-Instruct 或同系列 API
* **视觉语言模型 (VLM)**：Qwen2.5-VL-7B-Instruct
* **前端交互**：Gradio 提供 Web UI

## 🏗️ 系统架构

系统主要采用“资料上传 -> 任务路由 -> 直接VLM或动态索引检索 -> 大模型回答 -> 引用展示”的架构：
1. **资料上传与解析**：将 PDF 切分为单页，提取可复制文本并将每页渲染成图片。
2. **任务路由**：判断资料规模，决定是单图直接问答还是多页检索。
3. **临时索引与检索**：针对当前长文档构建 FAISS 索引，检索最相关的 Top-k 文本块或页面。
4. **生成回答**：结合用户问题与检索到的证据，调用 LLM 或 VLM 生成带有页码引用的最终回答。

## 🚀 快速开始

本项目推荐在普通电脑或云服务器上运行。本地推理方案需 1张 24GB GPU，可运行 Qwen2.5-VL-7B-Instruct 的量化推理。

    # 步骤 1. 克隆仓库
    git clone https://github.com/YourUsername/Course-Doc-VLM-QA.git
    cd Course-Doc-VLM-QA

    # 步骤 2. 安装依赖
    pip install -r requirements.txt

    # 步骤 3. 配置环境变量
    # 在项目根目录创建 .env 文件，并填入你的模型 API 密钥。

    # 步骤 4. 运行服务
    python app.py

## 📝 评测与实验

本项目计划在公开数据集（DocVQA或ChartQA子集）和自建中文课程文档集上进行评测。主要对比直接多模态问答、纯文本检索问答与页面级多模态检索问答的效果差异。
